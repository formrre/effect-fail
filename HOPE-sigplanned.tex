\begin{abstract}
Algebraic effects are one approach to tackling modularity issues
arising in structuring side effects in computation.
%applications where normally we would use monad transformers.
One such application area is in (formalised) programming language
semantics. This setting motivates extending algebraic effects
with a notion of non-recoverable failure.
%We show that the standard incarnation of effects does not satisfy some requirements needed to build compact,
%non-repetitive semantics, when the language does not have a type system such as in cases of intermediate languages in compilers.
We show how the MaybeT monad transformer can be used, studying the
implications on the underlying theory via a (work in progress)
mechanization in type
theory. Along the way we also give a graded monadic view of algebraic
effects.
\end{abstract}

\section{Motivation}

Consider the situation of a writing a verified compiler with a
reference interpreter where initial typing information has been
erased. This necessitates some error handling cases which are known
to not occur (assuming that the typing procedure for this language is sound):
%
\begin{haskell}
... UnOpNegate e -> case eval env e of
                      Num n -> Num (- n)
                      _     -> error "Impossible"
\end{haskell}
%
The \haskin{error} expression here represents
a failure side-effect that is intentionally non-recoverable; once it
is evaluate failure prevents the rest of the interpretation.

In Haskell, the above code pattern can be abstracted via
the \emph{do}-notation and the use of
the \haskin{MonadFail} type class which provides an operation
\haskin{fail :: String -> m a} which is used to desugar pattern matches which may
fail. In this style, the above code could be written as:
%
\begin{haskell}
... UnOpNegate e -> do
         Num n <- eval env e
         return (Num (- n))
\end{haskell}
%
Since the \haskin{(Num n)} pattern is `refutable' this
desugars into:
%
\begin{haskell}
  eval env e >>= (\v -> case v of
      Num n -> return (Num (- n))
      _     -> fail "Incomplete pattern match")
\end{haskell}
%
In this work, we capture the same idea but in the
setting of \emph{algebraic effects}, extending algebraic
effects to include a notion of (non-recoverable) failure
as captured by \haskin{MonadFail}.

Our development is formalised in dependent-type theory
(we use Agda) capturing
the mathematical results for the key
constructions. As part of this, we contribute our own
type theoretic rendering of free monads (based on a variation of
\citet{freerm}'s `freer' monads) which is indexed by the effect
operations captured within the computation tree describe by a free
monad value. Our approach yields
a graded monad, which provides a way to reason about the compositional
nature of effects with non-recoverable failure.

\section{Underlying model for algebraic effects}
%\dnote{functor strength}
Algebraic effects in theory consist of a signature of operations and
their equations. The collection of operations can be represented
type theoretically by a parameteric data type $F$ combined with the free monad
construction ($\mu X . A + F X$) to capture computations trees of
effect operations. However, this definition is not amenable to
formalisation in type theory because it leads to type-level fixed
points which leads to inconsistency. Most tools disallow this free
monad definition via positivity checks.

However, the free endofunctor generated by the left Kan extension
on any type constructor avoids this positivity issue. This
is captured by~\citet{freerm}'s `freer' monad construction, defined
(in Haskell) as:
\begin{haskell}
data FFree (f :: Type -> Type) (a :: Type) where
  Pure   :: a -> FFree f a
  Impure :: f x -> (x -> FFree f a) -> FFree f a
\end{haskell}
%
The \haskin{Impure} constructor captures effectful operations
described by the type constructor \haskin{f}.

In Agda, we use the same construction (plus
universe levels) but extended to allow a compositional
description of the effect operations captured within:
%
\begin{agda}[mathescape]
data Eff (E : List (Set -> Set)) (A : Set) : Set$_1$ where
  Pure   : A -> Eff E A
  Impure : {f : Set -> Set} {X : Set} {prf : f $\in$ E}
        -> f X -> (X -> Eff E A) -> Eff E A
\end{agda}
%
This provides a way to embed an effect to a list consisting of leafs results and effects, which need to be interpereted, consisting of the request F from a set E,
expecting a return type X and a continuation, which tells us how how to continue once handler provides a value X. This construction can be thought as a generalization of construction to provide a possibility to include multiple effects from a particular set.

\paragraph{Algebraic effects as monads and graded monads}

Monad on endofunctor $M$, consists of two natural transformations,
return, $\eta : 1 \rightarrow M$, and join, $\mu : M \circ M \rightarrow M$.
Graded monad consists of natural transformations, where Ms are annotated by the elements of a monoid, return being annotated by unit of the monoid
and join annotated by the multiplication of elements annotating the parts of composition.

Algebraic effects can be presented on top of a monadic interface, we provide this interface by implementing it using a free monad.
The purpose of this is the ability to chain different operations abstractly, postponing the concrete implementation until the full specification is provided.

For graded monads this allows us to do an interface, which combines
different effect sets, by representing them as lists.
This gives the following variants of graded monad operations, $\eta : 1 \rightarrow M []$ and $\mu : M e \circ M f \rightarrow M (e +\!\!\!+ f)$.

\paragraph{Handling effects}

%Handlers of effects can be thought of as a generalization of exception handlers, which provide richer context
%and are constrained to respect a certain set of equations.
Algebraic effects
provide ways to write programs causing effects which are treated
abstractly initially but which are given concrete implementations
afterwards using \emph{handlers} (akin to raising an exception and
later handling it).
This generalizes to general effects: effect handlers provide a way to
not only handle general exceptions, but to also remember some state
between handling different operations a handler is responsible for.
Handlers are then also constrained to respect a set of domain-specific
equations.
The graded monad construction suggests the following
set of combinators for raising effects,
handling effects and `escaping' from
an effectful computation once all effects have been
handled:
%
\begin{agda}
raise  : F a -> Eff [F] a
handle : ($\forall$ x -> F x -> x) -> Eff E a -> Eff (E - F) a
escape : Eff [] a -> a
\end{agda}
%
where \haskin{E - F} deletes \haskin{F} from the list
of effects. Note the grades give an account of what
effects needs handling, manipulated and leveraged
by these operations.


\section{Leveraging MaybeT and MonadFail}

In Haskell, if a \emph{do}-expression performs a pattern
matching `bind' that may fail, then the monad being used
must also have an instance of the \haskin{MonadFail} class:
%
\begin{haskell}
class MonadFail m where
  fail :: String -> m a
\end{haskell}
%
where, for all monadic computations \haskin{x :: m a}:
%
{\small{
\begin{equation*}
\hspace{-10em}\text{\haskin{fail}} >>= \text{\haskin{x}} \;\; \equiv \;\; \text{\haskin{fail}}
\end{equation*}
}}
%
i.e., failure is global and non-recoverable.

We have given this a more abstract mathematical definition,
ignoring the \haskin{String} aspect in favour of a map
from a terminal object $1$.
%
\begin{definition}[Failing monad]
  For a category $\mathcal{C}$ with terminal object $1$
  then a \emph{failing monad} for endofunctor $T : \mathcal{C}
  \rightarrow \mathcal{C}$ is a monad with a natural transformation
  $\mathit{fail}_A : 1 \rightarrow T\ A$ such that for all $g : 1
\rightarrow T\ B$ then:
$$
\mu \circ T (g \circ !_A) \circ \mathit{fail}_A = \mathit{fail}_B
$$
\end{definition}

If we attempt to give a \haskin{MonadFail} instance for the free
monad, we get stuck immediately: the free monad, is the monad for
which exactly the monad equations hold and nothing more, i.e.,
not the extra \haskin{MonadFail} axiom above. Therefore, we seek
to have both the benefits of the free monad structure for
capturing computation trees (for the purposes of effect handling)
but also the property of non-recoverable failure captured
by \haskin{MonadFail}. The simplest construction to support the
above law generally is provided by the ``\haskin{MaybeT}''
\emph{monad transformer}~\cite{transformerpub}, which
in Haskell is typically defined as follows:
%
\begin{haskell}
newtype MaybeT m a = MaybeT (m (Maybe a))
\end{haskell}
%
That is, a \haskin{MaybeT} is a monad homomorphism, mapping
$M -$ to $M (1 + -)$.
%
%MaybeT is the monad transformer, i.e. a transformation ${T A : Set} -> Monad T -> Maybe T A -> Monad S$.
%$MaybeT m x = m (Maybe x)$
%
We can think of \haskin{MaybeT} as an extension by an arbitrary,
single option failure case. Applying this to the free monad
structure yields a lawful instance of \haskin{MonadFail}:
%
\begin{haskell}
  instance MonadFail (MaybeT m a) where
    fail _ = MaybeT (return Nothing)
\end{haskell}
%
%especially in the strucutre of
%a Free monad as Monad is Pure and fixed point of a functor with this identity case.

However, we still require \emph{freeness} of this structure in order
to yield a suitable semantic core for handling algebraic effects
(augmented with non-recoverable failure). We therefore ask,
\emph{is the above the ``free MonadFail'' structure?}.

We have to modify handling, we state a different form of initiality, as much as effects are inspired by initiality, we provide a type-theoretic
motivation
for our definition, to provide handling of effectfail, we interpret the results and embed them into Maybe and propagate the underlying failure.
This should be correct not by some appeal to category theory as initiality is normally stated, but by appealing to the API we would expect from our system.
We would excpect this API to propagate pattern matching failure. To run EffectFail we either want to extract the underlying result or propagate the failure of underlying Pure result.

\paragraph{Algebraic freeness}

A monad $T$ is \emph{algebraically free on an endofunctor $F$} is
the category of $T$-algebras (monad algebras) is isomorphic to the category of
$F$-algebras. An algebraically-free monad is itself a free monad~\cite{nlab:free_monad,kelly_1980}.
Therefore, in order to consider the notion of a ``free failing
monad'', we characterise the category of \haskin{MonadFail} algebras.

\begin{definition}[Monad algebras]
  For a monad on $T : \mathcal{C} \rightarrow \mathcal{C}$, the category of monad algebras if the category
  of $T$-algebras which are compatible with the monad operations. That
  is $T\textsf{Alg}_{\mathsf{mnd}}$ is a category with objects as
  pairs $(A, h)$ where $A \in |\mathcal{C}|$ and $h \in \mathcal{C}(T\
  A, A)$ satisfying compatibility
  with the monad operations: $h \circ \eta = id$ and $h \circ \mu = h
  \circ T h$.
\end{definition}

We extend this idea to to algebras of failing monads.

\begin{definition}[Failing monad algebras]
  For a %terminal object $1$ and
  failing monad $T : \mathcal{C} \rightarrow \mathcal{C}$, the
  category of its algebras
  $T\textsf{Alg}_{\mathsf{mndFail}}$ is a category of monad algebras, with objects
  $(A, h)$ where
  $h \in \mathcal{C}(T A, A)$ is compatible with the monad operations
  and there exists a morphism $h' : 1 \rightarrow A$ such that:

\section{Future directions}
We have constructed the \haskin{Eff} monad in Agda and verified the various
monad / graded monad laws. For practical purposes of reasoning there needs to be more than just a
monadic structure.
We want to reason about the local code using the eponymous algebraic
laws. We would like to attach equations the handlers should satisfy to
the monad.
One solution for attaching equations to a datatype are Quotient Inductive Types~\cite{altenkirch2016type}. We would like to use these for providing correct-by-contruction handlers,
This should abstract away from the concrete choices when reasoning, yet force us to prove satisfiability of the algebriac equations by some handler.
To extend the proofs from our current work to the quotient case we need to prove the type-theoretic formulation of algebraic freeness (discussed above) in order to prove that the free monad constructions preserve all
equations. We note that algebraically-free monads are free, and all free monads are algbraically-free in particular categories~\cite{nlab:free_monad},
this result should follow in homotopy type theory.
The reason for this being that these proofs assume the ambient category to be
locally small and complete, with homotopy type theory we should be able
to get all colimits and right Kan extensions, which seem to be the used consequences of these assumptions.

These lemmas apply only to the free monad with resepect to a forgetful functor
$Mnd(C) \rightarrow Endo(C)$, we suggest a generalization that should
extend these to the forgetful functor $Mnd(C) \rightarrow
(ob(C),C)$, where $(ob(C),C)$ is the category resulting from forgetting all the structure with respect to $Set \rightarrow Set$.

There is no obvious generalization, which would let us talk about algebras over an arbitrary $(ob(C),C)$, we thus take a type theoretic view to
extend the isomorphism in the definition of algebraic freeness to the freer construction.

The semantics of a type constructor $G$ is thought as an initial
object in some category $Alg(G)$ corresponding to its algebraic description.
Homotopy type theory generalizes this to quotient types (and other more general higher
inductive types), thus we should be able to present F-algebras, including their equations, using quotient types.
Informally left Kan extension, Lan(X), of a datatype X should give us an endofunctor, which just accumulates functions applied to it
without actually applying them postponing their interpretation.
We propose that
the algebras of $G$ are isomorphic to the algebras of the left Kan extension of $G$:
%
\begin{conjecture}[Endofunctor algebras of Left Kan extensions]
Given a type family $G : Set \rightarrow Set$ with a homotopy-initial algebra semantics $Alg(G)$, the family of $Lan(G)$-endofunctor algberas is isomorphic to $Alg(G)$.
\end{conjecture}
%
If this conjecture holds, then the freer monad T over X would have T-algebras isomorphic to the initial algebras of X,
this follows by chaining this conjecture with the statement of algebraic freeness.

We would expect the formal statement of this conjecture to be provable
in some appropriate container \cite{altenkirch2015indexed} or generic levitated presentation \cite{DBLP:conf/icfp/ChapmanDMM10},
because this would allow us to connect the $\mathsf{Lan}(G)$-algebra of an endofunctor with the algebra of the underlying datatype $G$.

Another direction of future research is to construct an isomorphism between continuations and the fast type-aligned queues of Kiselyov, quotiented by their composition,
thus providing a more efficient implementation of effects without the overheads to reasoning this entails as we focus on semantic constructions instead of executable code, this is not our primary concern.
We would expect this to then transport by univalence, but as discussed
in \cite{unitransport} we would need also some form of representation
independence coming from parametricity for this to be actually
efficient.
