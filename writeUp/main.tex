\section{Motivation}

Algebraic effects provide a good\dMnote{What is meant by good
  exactly?} construction to flexibly add new operations to a monad
without much overhead. These provide ways to send messages, which are
to be replied to\dMnote{This sounds like more an analogy than what
  they actually provide}. We also want to provide a way to match on
patterns and fail in a default way, if they are not satisfied. These
are important for combining the approach of monadic semantics, with
type system information represented by constructors.

\section{Proving that Effect monad is a monad}

\subsection{Standard construction}

Standard proof about Haskell implementation of algebraic effects,
tends to be conceived in 2 steps, constructing a non-strictly positive
Free monad and proving that it is a monad for any functor and
instantiating it with a Left Kan extension. For implementation these
tend to be fused together and instead of requiring the left Kan
extension argument is instantiated with the type, we just require that
it is instantiated with a type, which occurs within a type-level list
at a given index stored within the datatype. Being in type-level list
is then enforced through type class constraints. This construction
looks like the following with F required to occur in E:

\begin{lstlisting}
data Effect (E :: [ * -> *]) A where
    Pure :: A -> Effect E A
    Impure ::  F X -> (X -> Effect E A) -> Effect E A
\end{lstlisting}

\subsection{Our construction in Agda}

We are working in dependently typed setting without type classes and
as such we need to do multiple tweaks. Without some notion of
covariance, we have to do the instantiation above in a single step.
Also Agda does not provide powerful enough type class resolution
mechanism and this would be unsatisfyingly metatheoretic. We construct
our effect monad by tweaking the Haskell construction slightly and
instead of providing an index to type-level list, we take a proof of
contained effect's membership in this list. As proofs of membership in
Agda are proof-relevant this at the same time acts as an index.

\section{Maybe transformer gives an instance of MonadFail}

We prove that the Maybe transformer is a monad instance for any monad
and we also proof that it is an instance of MonadFail for any monad.
The first one introduced together with monad transformers
\cite{transformerpub}, the second one through various Haskell
implementations \cite{mtl}. We give these constructions in Agda and
verify the laws.

\subsection{Effect API through Maybe transformer}

We lift the Effect API through Maybe transformer by identifying
sending with embedding Just values inside Effect, this has the desired
effect of being able to handle them properly. We then handle in the
same fashion. Once all effects are handled and the list of
possibilities is empty we have to extract the pure value, to do this
we note that impure case requires proof of membership inside the list
and thus it's not reachable, thus we extract the optional result from
Pure.
