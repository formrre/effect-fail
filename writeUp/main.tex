\subsection{Standard construction}

The standard Haskell implementation of algebraic effects
tends to be conceived in two steps: constructing a non-strictly positive
\haskin{Free} monad and proving that it is a monad for any functor,
then instantiating it with a Left Kan extension.
\dnote{show it and explain more}
For an implementation, these two steps
tend to be fused together and instead of requiring the left Kan
extension argument is instantiated with the type, we just require that
it is instantiated with a type, which occurs within a type-level list
at a given index stored within the datatype.

 Being in type-level list

\section{Maybe transformer gives an instance of MonadFail}

\dnote{needs intro text, e.g.}
A standard approach of adding a notion of failure to an existing
monad is via a \emph{monad transformer}, typically
\haskin{MaybeT}~\cite{transformerpub}:
%
\begin{haskell}
newtype MaybeT m a = MaybeT { runMaybeT :: m (Maybe a) }
\end{haskell}
%
We prove that this transformer is a monad for any monad \haskin{m}.
We also proof that it is a lawful instance of \haskin{MonadFail} for
any monad \haskin{m}.
The first one introduced together with monad transformers
\cite{transformerpub}, the second one through various Haskell
implementations \cite{mtl}.\dnote{not sure what this part was saying,
